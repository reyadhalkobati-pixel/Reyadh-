# 📱 Android APK — qabout-file

This project is wrapped with **Capacitor** and ships with a one-click APK
build pipeline. You have **three** ways to get a real installable `.apk`.

App ID: `com.alborihi.qaboutfile`
App Name: **qabout-file**

---

## 🚀 Option 1 — Cloud build (zero local setup, recommended)

1. Create a new empty GitHub repo (private or public).
2. Push this entire project to it:
   ```bash
   git init && git add . && git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/<you>/<repo>.git
   git push -u origin main
   ```
3. Open the repo on GitHub → **Actions** tab → wait ~5 min.
4. Click the latest run → download the **`qaboutfile-debug-apk`** artifact.
5. Install the `.apk` on your Android device (enable "Install unknown apps").

The workflow lives at `.github/workflows/android-apk.yml` and runs on
every push.

---

## 💻 Option 2 — Local build

Requirements: **Node 20+**, **JDK 17**, **Android SDK** (Android Studio installs both).

```bash
bash scripts/build-apk.sh
```

APK output: `android/app/build/outputs/apk/debug/app-debug.apk`

## 🌐 Option 3 — PWABuilder (fastest, deployed PWA only)

If your web app is already deployed as a PWA:

1. Open <https://www.pwabuilder.com/>
2. Paste your live URL.
3. Click **Package for stores → Android → Download**.

---

## Signing for Play Store (optional)

The bundled workflow produces a **debug-signed** APK suitable for sideload.
For a release build / Play Store upload:

1. Generate a keystore: `keytool -genkey -v -keystore release.keystore -alias key -keyalg RSA -keysize 2048 -validity 10000`
2. Add it as a GitHub secret + extend the workflow with `./gradlew assembleRelease`.
